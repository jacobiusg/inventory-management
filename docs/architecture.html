Data Flow — Filtered Orders Request

1
View
Orders.vue mounts and calls loadOrders().
2
Composable
useFilters singleton holds period, location, category, status as Vue refs.
3
API Client
api.getOrders(filters) builds query string, drops all* values.
4
HTTP
Axios → GET http://localhost:8001/api/orders?month=2025-05&warehouse=Tokyo&status=Shipped
5
FastAPI
Route handler validates params, calls apply_filters() + filter_by_month().
6
Data
Filters in-memory list loaded from server/data/orders.json.
7
Pydantic
Validated Order models serialized to JSON.
8
Render
Vue updates refs; computed properties drive table rows, badges, totals.

Filter System

Time Period

  • Format YYYY-MM or Q#-2025
  • Applied via filter_by_month()
  • Inventory ignores time (no time dimension)

Warehouse · Category · Status

  • Applied via apply_filters()
  • Sent as query params on every fetch
  • Shared state lives in useFilters

Propagation

  • All views subscribe to the same singleton
  • Watchers refetch when any filter changes
  • Reset button clears all four at once
